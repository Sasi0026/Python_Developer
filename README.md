## 1. ArXiv Research Paper Scraper

🔗 **Repository:** [Sasi0026/ArXiv_Research_Papers_Scraper](https://github.com/Sasi0026/ArXiv_Research_Papers_Scraper)

### Overview
A Streamlit-based web application that automates the discovery and organization of academic research papers from arXiv.org. Users can search by topic, control the number of results, and browse structured paper metadata — all without touching the arXiv website manually.

### Problem Solved
Manually browsing arXiv for relevant research is time-consuming and unstructured. This tool centralizes the search, filters results by topic, and presents them in a clean, interactive UI.

### Key Features
- 🔍 Topic-based search with configurable result count
- 📄 Displays paper title, authors, abstract, and direct arXiv link
- ⚡ Real-time scraping on search trigger
- 🖥️ Clean Streamlit UI — no frontend knowledge required to run

### Tech Stack
| Tool | Purpose |
|------|---------|
| Python | Core logic |
| Streamlit | Web UI framework |
| BeautifulSoup | HTML parsing |
| Requests | HTTP calls to arXiv |

## 2. SprintRunner — OOP Timer App

🔗 **Repository:** [Sasi0026/SprintRunner](https://github.com/Sasi0026/SprintRunner)  
🔄 **Status: In Progress**

### Overview
A Python-based productivity timer application built entirely using Object-Oriented Programming principles. SprintRunner allows users to define a total task duration, divide it into focused work blocks, and triggers an alarm at the completion of each block — keeping the user on track without manual clock-watching.

### Problem Solved
Context switching and losing track of time during deep work sessions. SprintRunner enforces structured work intervals programmatically, similar to the Pomodoro technique — but fully customizable and code-driven.

### Key Features
- ⏱️ Configurable total task time and work block size
- 🔔 Alarm trigger on each completed work block
- 🏗️ Fully designed with OOP — classes for Timer, Block, and Alarm
- 🔄 Tracks remaining blocks and elapsed time

### Tech Stack
| Tool | Purpose |
|------|---------|
| Python | Core application logic |
| OOP (Classes) | Timer, Block, Alarm architecture |
| Threading *(planned)* | Non-blocking alarm execution |

> 🚧 Active development — features being added. Contributions and feedback welcome.

## 3. Google Search Automation

🔗 **Repository:** [Sasi0026/google-search-automation-selenium](https://github.com/Sasi0026/google-search-automation-selenium)

### Overview
An automated bulk Google search tool built with Python and Selenium WebDriver. Reads a list of queries from a text file and executes them programmatically across multiple browser tabs — with anti-bot detection techniques to ensure reliable execution.

### Problem Solved
Repetitive manual searching of multiple queries for research or assignments. This tool processes 10+ searches in under 30 seconds, saving 2–3 hours per week of manual effort.

### Key Features
- 📂 File-based input — reads queries from `questions.txt`
- 🗂️ Multi-tab automation — each result opens in a new tab
- 🛡️ Anti-bot detection — custom user-agent + WebDriver stealth via CDP
- 🌐 Cross-browser support — Chrome and Edge compatible

### Tech Stack
| Tool | Purpose |
|------|---------|
| Python | Core scripting |
| Selenium WebDriver | Browser automation |
| Chrome DevTools Protocol (CDP) | Anti-detection configuration |

### Results
- ✅ 10+ searches automated in under 30 seconds
- ✅ Reusable framework for any bulk search scenario
